# Enterprise AI Readiness Assessment — Project Context

Handoff document for resuming development across sessions/accounts.

---

## What This App Is

A **consultant-facing tool** built for Logic2020 to conduct structured AI readiness assessments with enterprise clients. A consultant conducts an interview with a respondent (executive, manager, or practitioner), scores 60 questions across 5 dimensions on a 1–5 scale, and the tool generates a scored report with recommendations.

**There is no AI/ML powering the scoring.** The name refers to the subject matter (AI readiness), not the technology. All scoring is deterministic math in the browser. No backend, no API, no database.

## How Scoring Works

1. Consultant rates each question **1–5** (or marks **?** = Don't Know) during interview
2. Scores normalized to **0–100**: `((sum - n) / (n × 4)) × 100` where `n` = answered questions
3. DK answers excluded from scoring, tracked separately as a visibility indicator
4. Composite = unweighted mean of individual session scores per dimension
5. Recommendations generated by tier logic in `src/data/recommendations.js` (no ML)
6. Narrative in `src/data/questions.js` → `generateNarrative()` is template text based on score ranges

## Assessment Structure

- **5 Dimensions:** Strategy (1), Data (2), Governance (3), Talent (4), Operations/MLOps (5)
- **12 questions per dimension** = 60 total
- **Scale:** 1 (Not at all) → 5 (Advanced), plus **?** (Don't Know / excluded)
- **Role Groups:** Executive, Management, Practitioner — drive perception gap analysis

## Key Architectural Decisions

### `avg: null` pattern
Dimensions with all-DK responses return `avg: null` (not 0). All rendering paths null-guard with `d.avg ?? '—'`. Radar chart excludes null-avg dimensions entirely.

### `DK = 'dk'` constant
Exported from `questions.js`. A DK answer is excluded from dimension scores and tracked as a visibility indicator. High DK rates surface as a "Low Organizational Visibility" callout.

### `computeComposite(sessions)` in engagement.js
- Returns full composite object with per-dimension `avg`, `stdDev`, `byGroup`, `qAvgs`, `dkRate`, `lowVisibility`, `allDK`, `perceptionGap`, `gapSeverity`
- `singleRoleGroup` flag warns when gap analysis isn't meaningful (all respondents same group)
- `sessionsByGroup` cached before the dimension loop (performance)

### `scoreTier(score)` in constants/thresholds.js
Single source of truth for maturity tiers. Used by recommendations.js and questions.js. Thresholds: `< 40` = low, `< 65` = medium, `>= 65` = high.

### Storage
- Uses `localStorage` with quota monitoring
- `STORAGE_QUOTA_BYTES = 5_242_880` (5MB)
- Fires `ai_storage_warning` (70%) and `ai_storage_full` events
- App.jsx listens and renders `StorageWarningBanner` with export CTA

### PDF Export
- jsPDF draws all text/tables as vector
- html2canvas screenshots the radar chart only
- On canvas failure: renders placeholder page + amber warning banner to user
- `HTML2CANVAS_SCALE = 1.5` in thresholds.js

## Data Flow

```
Interview (consultant scores questions)
    ↓
buildSession() — validates, normalizes, computes dimScores + overallScore
    ↓
addSession(engagement, session) — merges into engagement object
    ↓
saveEngagement() — persists to localStorage + fires storage quota checks
    ↓
exportSession() — auto-downloads JSON backup after save
    ↓
computeComposite(sessions) — aggregates all sessions into composite dims
    ↓
CompositeResults — renders report, radar, recommendations, PDF export
```

## Role Group Auto-Detection

`assignRoleGroup(roleTitle)` in `engagement.js` uses keyword matching:
- **Executive:** CEO, CTO, CIO, CDO, VP, Director, President, Chief…
- **Management:** Manager, Lead, Head, Architect, Supervisor…
- **Practitioner:** Engineer, Analyst, Scientist, Developer, Specialist… (default fallback)

Consultant can override via the role group selector on `RespondentForm`.

---

## Perception Gap Analysis

- Flagged when any two role groups diverge by **≥ 15 points** on a dimension (`PERCEPTION_GAP_THRESHOLD`)
- Severity tiers: **concerning** (15–25 pts), **severe** (25–40 pts), **critical** (40+ pts)
- `PerceptionGapCallout` in CompositeResults renders severity-specific interpretation copy
- `singleRoleGroup` composite flag suppresses the callout with an explanation

## Industry Intelligence

- `industryProfiles.js` — qualitative maturity tiers, compliance risk flags, Gov/regulated industry detection
- `isGovPhase1Industry()` — moves Governance to Phase 1 in the sequencing recommendation for regulated industries (Healthcare, Financial Services, Government, Energy)
- **No numeric benchmark data** — removed because there was no valid public source. Logic2020's own assessment data is the path to real benchmarks (tracked in memory).

---

## Investment Sequencing Logic

Fixed 4-phase roadmap in `CompositeResults.jsx`:
1. **Phase 1 — Foundation:** Strategy (prerequisite for all)
2. **Phase 2 — Enablement:** Data + Governance (parallel track)
3. **Phase 3 — Delivery:** Operations/MLOps
4. **Phase 4 — Scale:** Talent (org-wide AI enablement)

For regulated industries, Governance is elevated to Phase 1 alongside Strategy.
